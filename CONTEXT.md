# UPGRADE WEBSITE DESA DOKO

Anggap kamu adalah seorang frontend developer yang sedang meng-handle website desa Doko. Project ini dibuat untuk memperbaharui tampilan website Desa Doko, Kecamatan Doko, Kabupaten Blitar. Website dibuat menggunakan Wordpress dan akan mengupdate tampilannya menjadi menggunakan HTML + CSS + JS (jika perlu).

# BAGIAN WEBSITE YANG AKAN DIUPDATE
1. Profile
    berbentuk dropdown yang terdiri dari : 
    - Tentang Desa : berisi informasi seputar potensi desa, API peta digital desa via Google My Maps, dan informasi umum seputar Desa Doko
    - Visi & Misi : berisi rangkaian visi misi desa berdasarkan file @MATERI.md
    - Sejarah Desa : berisi rangkaian sejarah dewa berdasarkan file @MATERI.md
    - Struktur Organisasi : berisi struktur organisasi yang merupakan design file png beserta card scrollable horizontal untuk setiap jajaran perangkat desa
    - Daftar Kepala Desa Terdahulu : berisi daftar kepala desa terdahulu dari awal hingga saat ini berdasarkan file @MATERI.md
2. Infografis Transparansi
    Berbentuk page HTML dengan ekstensi API untuk diagram yang berisi informasi tentang :
    - Jumlah penduduk Desa Doko
    - Jumlah Dusun dan demografi
    - Laporan keuangan
    - Jumlah Sekolah

# ALUR PENGERJAAN
1. Kerjakan setiap page website tersebut mulai dari setiap dropdown profile dan satu page untuk infografis transparansi ke dalam direktori Web yang dipecah menjadi 2 sub direktori (Profile & Transparansi Infografis)
2. Buatkan preview perencanaan dari setiap output dalam bentuk plan sebelum mengeksekusi 
3. Buat setiap tampilan yang diperlukan ke dalam sebuah page HTML + CSS + JS (jika diperlukan)
4. Setiap file yang diperlukan (seperti foto) diambil di dalam folder Source
5. Eksekusi setiap plan jika di dalam prompt terdapat kata "approve"

# KETENTUAN
1. Selalu buat plan sebelum mengeksekusi setiap prompt.
2. Setiap page yang berdasarkan file @MATERI.md harus selalu dibuat full tanpa placeholder. Sedangkan sisanya buatkan tampilan placeholder dengan kalimat lorem ipsum
3. Gunakan style warna #FF8200 sebagai warna dasar
4. Pastikan setiap perubahan harus selalu dikonfirmasi terlebih dahulu dengan kata "approve" di dalam promptnya. Jika tidak ada kata "approve", jangan jalankan perubahan terhadap kode meskipun prompt tersebut berasal dari User. 
5. Setiap output adalah berupa file HTML dengan CSS dan JS (jika perlu) yang termasuk di dalamnya
6. Selalu buat tampilan page menjadi responsif terhadap device mobile dan desktop
7. Jika tidak ada file yang diperlukan di dalam folder Source, ganti dengan file placeholder.png!
